Grid container

Это набор пересекающихся горизонтальных и вертикальных grid линий, которые делят пространство grid контейнера на grid области, в которые могут быть помещены grid элементы. Внутри grid контейнера есть два набора grid линий: один определяет ось столбцов, другой определяет ось строк.

Grid lines

Горизонтальные и вертикальные разделители grid контейнера. Эти линии находятся по обе стороны от столбца или строки. Нумерация начинается с единицы. Важный нюанс, данный элемент восприимчив к режиму написания, который используется на вашем ресурсе.

Grid track

Пространство между двумя смежными grid линиями, вертикальными или горизонтальными.

Grid cell

Наименьшая неделимая единица grid контейнера на которую можно ссылаться при позиционировании grid элементов. Образуется на пересечении grid строки и grid колонки.

Grid area

Пространство внутри grid контейнера, в которое может быть помещен один или больше grid элементов. Этот элемент может состоять из одной или более grid ячеек.

Defining a grid

display: grid - объявляет элемент как Grid. Все непосредсвенные дети становятся Grid Items.

grid-template-columns и grid-template-rows - объявляют явную сетку (Explicit Grid).

grid-gap - задает пространство между Tracks.

Fractions

Fraction (fr, доля, фракция) - представляет долю доступного пространства в grid-контейнере для Track. Похоже на flex-basis. 

      grid-template-columns: 1fr 1fr 1fr;
      grid-template-columns: 400px 1fr 1fr;

Repeat

repeat - позволяет задавать повторяющиеся патерны из Tracks.

grid-template-columns: repeat(3, 1fr);

Implicit grid

Явная сетка это та, которую мы указываем в CSS. Неявная сетка это та, которую CSS grid создаст автоматически для того, чтобы вместить контент.

К примеру если мы явно не укажет grid-template-rows, CSS grid автоматически создаст горизонтальные Tracks под контент. Эти Tracks будут иметь автоматические размеры, зависящие от контента.

grid-auto-rows и grid-auto-columns задают размеры для не явно созданых Tracks.

auto-fill & minmax

repeat(auto-fill, 200px) - говорит контейнеру о необходимости создать любое кол-во Tracks определенного размера.

repeat(auto-fill, minmax(200px, 1fr)) - создает респонсив grid. Первый аргумент - минимальный размер Track, второй - максимальный. Если второй аргумент равен 1fr, Track займет доступное пространство.
